Walk-through

This mirrors the guided explanation style in the screenshots, but in a cleaner web layout.

1) Start at the Dashboard

Use the dashboard as your command center. It provides one-click access to each database and subpage.

2) Use database views

Each area is treated as a database: Projects, Notes, Literature, Papers, Thesis, Career. Keep each page updated weekly.

3) Link everything together

As in your template, tasks connect to papers and chapters, so progress is visible both by project and by deadline.

4) Work paper-by-paper and chapter-by-chapter

Open each specific paper/chapter page for detailed sections, protocols, figures, notes, and writing status.

5) Keep career pages live

Publication record, opportunities, and contacts act as your outward-facing planning layer.

Edit in GitHub (No CLI)

  1. Open any linked page above.
  2. Click Edit this file (pencil icon).
  3. Update table rows and notes.
  4. Commit your changes directly in GitHub.
